# SuveyTool4MBA V2.0 系统说明报告

## 一、系统概述

SuveyTool4MBA V2.0 是一个面向 DBA / MBA 用户的商业决策验证工作流工具。

它不是传统意义上的“统计计算器”，而是一个从业务痛点出发，逐步完成研究建模、问卷设计、数据体检、统计分析与管理层解释输出的辅助系统。

当前版本的核心定位是：

- 浏览器本地分析
- 外部 AI 提示词协作

也就是说，系统当前已经能够完成主要研究与分析流程，但在部分 AI 生成环节，仍采用“系统生成提示词，用户复制到外部 AI 工具协同完成”的方式，而不是完全站内自动化。

## 二、目标用户

本系统主要适用于以下用户：

- 具有丰富业务经验、但缺乏统计学基础的管理者
- DBA / MBA 学习者
- 希望把业务直觉转化为结构化研究问题的用户
- 需要用量化分析辅助商业决策的非技术用户

## 三、系统适合解决的问题

当用户面对以下问题时，可以使用本系统：

- 某项管理措施为什么效果不佳
- 某种组织、营销或数字化策略是否真正有效
- 某个业务问题背后到底有哪些关键影响因素
- 如何把主观判断转化为可验证的量化研究模型

本系统的价值，在于帮助用户从“业务问题”走向“研究验证”，并输出更适合管理层理解的解释结果。

## 四、当前版本主要功能

### 4.1 首页与项目入口

系统首页支持两种进入方式：

- 新建我的项目
- 查看示例项目

用户既可以从零开始搭建自己的研究项目，也可以先通过示例项目理解系统整体工作流。

### 4.2 故事转模型

用户可以用自然语言描述自己的业务痛点、研究兴趣或管理疑问。系统会基于这些输入生成一段专业提示词，用于外部 AI 助手。

当前流程为：

1. 用户输入业务背景
2. 系统生成专业提示词
3. 用户将提示词复制到外部 AI 工具，例如 ChatGPT、DeepSeek、通义千问
4. 用户把 AI 返回的 JSON 粘贴回系统
5. 系统自动解析并生成研究标题、研究问题、构念、假设与题项

这部分功能已经可用，但当前仍属于“AI 提示词协作”模式，不是站内 API 自动化模式。

### 4.3 研究设计工作流

系统支持围绕研究项目进行完整的结构化编辑，包括：

- 项目总览
- 研究问题定义
- 构念管理
- 假设管理
- 问卷题项管理

用户可以在系统内逐步完善研究设计，而不必直接面对统计软件中的复杂配置。

### 4.4 发布与收集

系统当前提供一个“发布与收集”步骤，用于承接未来的在线问卷发布能力。

当前已包含：

- 问卷状态展示
- 截止日期设置
- 分享链接展示
- 收集进度展示

目前这一部分更偏向工作流占位与演示能力，还不是一个完整的在线问卷收集平台。

### 4.5 红绿灯数据体检

这是当前版本的重要功能之一。

用户上传 CSV 数据后，系统会在浏览器本地执行数据质量检查，并用红绿灯方式反馈结果。

当前已实现：

- CSV 数据上传
- Cronbach's Alpha 计算
- AVE 计算
- 红绿灯质量反馈
- 拖后腿题项识别
- 剔除题项后重新计算

相比传统统计软件直接展示大量数值，这一模块更强调“是否可用”与“该怎么改”。

### 4.6 数据分析

系统当前支持基于现有模型执行 PLS-SEM 分析，并输出结构化分析结果。

当前已支持：

- 本地执行分析
- 展示路径系数
- 展示 R²
- 展示 Bootstrap 结果
- 路径图与图表展示

系统会尽量以更适合业务用户理解的方式组织这些结果。

### 4.7 董事会报告辅助生成

当分析完成后，系统会根据结果生成适合外部 AI 使用的提示词，帮助用户获得更偏管理层语言的报告文本。

当前流程为：

1. 系统整理分析结果
2. 系统生成董事会报告提示词
3. 用户复制提示词到外部 AI 助手
4. 外部 AI 生成更完整的高管风格报告

因此，这一模块当前可理解为“董事会报告辅助生成”，而非“系统站内自动报告生成”。

## 五、推荐使用流程

对于第一次使用系统的用户，建议按以下步骤操作：

1. 打开首页
2. 点击 `新建我的项目`
3. 在 `AI 建模助手` 中输入业务痛点
4. 复制系统生成的提示词
5. 将提示词粘贴到外部 AI 工具
6. 把返回的 JSON 结果粘贴回系统
7. 应用结果，生成项目结构
8. 检查并修改构念、假设与问卷题项
9. 上传 CSV 数据
10. 执行 `数据体检`
11. 如果出现红灯，先优化题项质量
12. 进入 `数据分析`
13. 在 `研究报告` 步骤中复制提示词到外部 AI，生成更完整的高管风格结论

如果只是想快速理解系统，也可以先点击：

- `查看示例项目`

通过示例项目快速浏览整体功能。

## 六、当前版本边界

当前版本尚未实现以下能力：

- 站内 OpenAI / DeepSeek API 自动调用
- 用户 API Key 设置面板
- 自动统计算法路由（T-test / ANOVA / Chi-square）
- 在线问卷收集系统
- 团队协作
- 多组分析
- 中介 / 调节专项分析

这些内容属于未来版本的开发计划，不应与当前已交付功能混淆。

## 七、数据与隐私说明

当前版本强调本地处理与数据安全：

- 原始业务数据主要在浏览器本地处理
- 系统不会自动把原始数据上传给外部 AI
- 只有用户主动复制出去的提示词内容，才会进入外部 AI 对话

这意味着在当前架构下，企业原始数据的敏感性得到了较好的控制。

## 八、当前版本一句话总结

SuveyTool4MBA V2.0 当前可以被准确概括为：

**一个以浏览器本地分析为基础、以外部 AI 提示词协作为补充的商业决策验证工作流系统。**
